# 도로보GO 개발 지침

## 지원 플랫폼

도로보GO는 PC 웹앱과 모바일 앱을 모두 지원한다.

기능을 추가하거나 수정할 때 사용자가 별도로 말하지 않아도
웹앱과 모바일 앱에 미치는 영향을 함께 검토한다.

- 웹앱: 관제, 사건 판단, 출동 배정, 사용자 및 시스템 관리
- 모바일 앱: 출동 요청 확인, 수락, 이동, 현장 도착, 조치 완료 보고

## 구현 원칙

1. 사용자가 웹앱 또는 모바일 앱 한쪽만 지정하면 해당 플랫폼만 수정한다.
2. 플랫폼 지정이 없으면 웹앱과 모바일 앱 모두에 필요한 기능인지 먼저 판단한다.
3. 양쪽에서 필요한 기능은 공통 타입, API, 상태값을 재사용한다.
4. 웹앱과 모바일 앱이 같은 서비스로 인식되도록 로고, 컬러, 상태 표현을 통일한다.
5. 기존 파일 구조와 컴포넌트를 먼저 확인하고 중복 코드를 만들지 않는다.
6. 기존 기능과 디자인을 임의로 삭제하거나 크게 변경하지 않는다.

## 7. 사용 원칙

- 배경은 **밝게 유지**하고 정보는 **명확하게 구분**한다.
- 파랑과 청록은 서비스의 **핵심 브랜드 경험**을 만든다.
- 주황, 빨강, 초록은 **상태 표현용**으로만 제한적으로 사용한다.
- 흰색(`#FFFFFF`)은 **모달, 입력창, 최상위 팝업** 정도에만 최소 사용한다.
- 전체적으로 **너무 어둡지 않고**, **전문적이고 신뢰감 있는 공공안전 UI** 톤을 유지한다.
